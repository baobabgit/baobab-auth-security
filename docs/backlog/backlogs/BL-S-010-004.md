# BL-S-010-004 — Implementer refresh tokens opaques

:version_target: v0.1.0
:priority: P1
:depends_on: [BL-S-010-001]

## Contexte

Générer des refresh tokens opaques aléatoires, leur hash stockable et leur
identifiant, avec dates UTC. Le clair n'est rendu qu'une seule fois ; security
ne persiste rien.

## FEAT liées

- FEAT-010.3 — Refresh tokens opaques (US-010)

## Critères d'acceptation

- [ ] `RefreshTokenHasher` (SHA-256, comparaison temps constant) implémenté.
- [ ] `RefreshTokenResult` porte `token` (masqué), `token_hash`, `token_id`,
      `issued_at`, `expires_at` (UTC aware).
- [ ] `RefreshTokenGenerator.generate` produit un token à forte entropie
      (`secrets`) ; deux générations diffèrent.
- [ ] Aucun token brut exposé dans `repr`/`str`.
