# BL-S-010-006 — Implementer adaptateurs core

:version_target: v0.1.0
:priority: P0
:depends_on: [BL-S-010-002, BL-S-010-003, BL-S-010-004, BL-S-010-005]

## Contexte

Brancher les services techniques sur les ports **réels** (synchrones) de
`baobab-auth-core` via des adaptateurs stricts et des mappers de value objects,
et fournir une révocation minimale par `jti`. Voir ADR-0005.

## FEAT liées

- FEAT-011.1 — Adaptateurs et mappers `baobab-auth-core` (US-011)

## Critères d'acceptation

- [ ] `InMemoryRevocationChecker` (révoque/teste un `jti`).
- [ ] `CorePasswordHasherAdapter` satisfait le port `PasswordHasher` réel.
- [ ] `CoreTokenProviderAdapter` satisfait le port `TokenProvider` réel
      (6 méthodes, `verify_*` → `dict`).
- [ ] `CoreClaimsMapper` (SecurityTokenClaims ⇆ TokenClaims du core, sans
      recalcul rôle→permissions) et `CoreTokenPairMapper` (→ TokenPair du core).
- [ ] `CoreRevocationAdapter` refuse un access token au `jti` révoqué.
- [ ] Tests d'intégration contractuels (`runtime_checkable`) verts.
